    我们来讲解一下Cache的一些注解

    @Cacheable

    配置在方法或类上，作用：本方法执行后，先去缓存看有没有数据，如果没有，从数据库中查找出来，给缓存中存一份，返回结果，下次本方法执行，在缓存未过期情况下，先在缓存中查找，有的话直接返回，没有的话从数据库查找，主要针对方法的注解，可以根据方法请求参数对返回值进行缓存。主要参数有

    • value （缓存的名称，必须指定至少一个,@Cacheable(value = “user”)或者@Cacheable(value = {“user”,”role”})）
    • key （可选，缓存的key，如果需要指定参数则需要使用SPEL表达式编写，否则默认使用方法所有参数组合，@Cacheable(value = “user”, key=”#username”)）
    • condition unless （缓存的条件，可以为空）

    • @CachePut

      也是针对方法使用，可以根据方法请求参数对返回值进行缓存，和@Cacheable不同的是每次都会触发真实的查询。类似于更新操作，即每次不管缓存中有没有结果，都从数据库查找结果，并将结果更新到缓存，并返回结果。
    • 参数和@Cacheable一样

    • @CacheEvict

      用来清除用在本方法或者类上的缓存数据，针对方法配置，根据一定的条件对缓存进行清空
    • value key condition等@Cacheable一样
    • allEntries 是否清空所有缓存内容，默认false
